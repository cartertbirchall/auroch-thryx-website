Built as infrastructure, not a single app.

Auroch is a hardware-flexible AI operating ecosystem. It can run locally where useful, route through edge and cloud where necessary, and expose specialized branches through a common operator and service contract.

Platform

Branch-based operating model

Each branch performs a distinct function while sharing contracts, manifests, persistence, and orchestration.

Distribution

Local, edge, and cloud

The architecture is not dependent on one vendor or device class. Compute is placed where the workload belongs.

Core branches.

The public framing foregrounds the investable operating ecosystem: Merc, AVN, Studio, DeMille, DreamWare, Prana, and Persistence.

Merc

Intent and routing

The decision layer that interprets requests and sends work to the right branch.

AVN

News intelligence

A plug-and-play information company for briefings, signals, and balanced reporting.

Studio

Artifact production

Turns briefings, scripts, and plans into usable media and production outputs.

DeMille

Cinematic intelligence

Shot planning, continuity, direction, and cinematic rendering logic.

DreamWare

Surreal media

The creative branch for stylized worlds, impossible media, and image/video experiments.

Prana

Operator interface

The control surface for the ecosystem — important, but not the whole company.

Persistence

Payloads, manifests, memory, cloud substrate

The durable layer that makes the system cumulative rather than disposable.

Auroch Thryx is not a chatbot skin. It is an operating ecosystem for routing intelligence, producing artifacts, directing media, and preserving institutional memory.
